## 0. Global Skill Sharing System

### SYNTHIA Skill Manager
- **Module:** `src/synthia/skill_manager.py`
- **Singleton:** `from src.synthia.skill_manager import get_skill_manager`
- **Purpose:** Centralized skill management with multi-source discovery, cross-tool synchronization, and security scanning.
- **Key methods:**
  - `skill_manager.search(query, sources, limit)` — Search skills from SkillsMP, SkillHub, ClawHub, or local
  - `skill_manager.install(skill_id, source, scope, target_tools)` — Install skill with security scanning
  - `skill_manager.sync(skill_name, target_tools)` — Sync skill across multiple AI tools
  - `skill_manager.list_installed()` — List all installed skills
  - `skill_manager.generate_report()` — Generate skill matrix report
- **Config flag:** `[FEATURES] SKILL_MANAGER_ENABLED = "true"` in `config.toml`
- **API Keys:** Set `SKILLSMP_API_KEY` environment variable for SkillsMP search (optional — SkillHub and ClawHub work without API key)

### Supported Skill Sources

| Source | API Key Required | Description |
|--------|------------------|-------------|
| **SkillsMP** | Yes (`SKILLSMP_API_KEY`) | Curated skills with AI semantic search |
| **SkillHub** | No | Community skills catalog |
| **ClawHub** | No | Versioned skills with semantic search |
| **Local** | No | Skills in local `skills/` directory |
| **n8n** | No | n8n workflow integrations |

### Skill Installation Scopes

- **GLOBAL** — User-level skills available across all projects
- **PROJECT** — Project-level skills only for current project

### Security Scanning

All skills are automatically scanned at install time for:
- **Critical:** Destructive commands, remote code execution, disk operations
- **High:** Hardcoded secrets, unsafe deserialization, SSL bypass
- **Medium:** Debug statements, TODO comments, empty blocks

### Quick Start Commands

```python
from src.synthia.skill_manager import get_skill_manager

# Get skill manager instance
sm = get_skill_manager()

# Search for skills

# Sync across tools

# Generate report

```

## 1. Security & Safety

### ACIP — Advanced Cognitive Inoculation Prompt Checker
- **Module:** `src/security/acip_integration.py`
- **Singleton:** `from src.security.acip_integration import acip`
- **Purpose:** Detects and blocks prompt-injection attacks, jailbreak attempts, and leaked destructive instructions before any LLM call.
- **Key method:** `acip.check_prompt(prompt: str) -> {"allowed": bool, "verdict": str, "sanitized": str}`
- **Config flag:** `[FEATURES] ACIP_ENABLED = "true"` in `config.toml`
- **Called automatically** in `src/llm/llm.py` before every model inference.
- **Patterns blocked:** `rm -rf`, `sudo rm/dd/mkfs/shutdown`, `IGNORE PREVIOUS INSTRUCTIONS`, `DISREGARD ALL PRIOR`, `open SSH`.

### DCG — Destructive Command Guard
- **Module:** `src/sandbox/dcg_wrapper.py`
- **Singleton:** `from src.sandbox.dcg_wrapper import dcg`
- **Purpose:** Intercepts shell/OS commands generated by Runner and Patcher agents and blocks destructive ones.
- **Key method:** `dcg.check_command(command: str) -> {"allowed": "yes"/"no", "reason": str}`
- **Config flag:** `[FEATURES] DCG_ENABLED = "true"` in `config.toml`
- **Called automatically** in `src/agents/runner/runner.py` before every `subprocess.run()` call.
- **Patterns blocked:** `rm -rf`, `rm -r`, `shutdown`, `poweroff`, `:>`, `dd if=`, `mkfs`.

## 2. Skill Registry

### Meta Skill (ms) Client
- **Module:** `src/integrations/ms_client.py`
- **Singleton:** `from src.integrations.ms_client import ms_client`
- **Purpose:** Progressive skill disclosure — search and fetch task-relevant skills from a local `ms` skill registry before building agent prompts.
- **Key methods:**
  - `ms_client.search_skills(query: str) -> List[Dict]` — returns ranked skill list
  - `ms_client.fetch_skill(skill_id: str) -> Dict` — fetch full skill detail
  - `ms_client.available() -> bool` — check if `ms` binary is on PATH
- **Config flag:** `[FEATURES] MS_BINARY = ""` — set path to `ms` binary to enable
- **Called automatically** in `src/agents/planner/planner.py` before rendering the plan prompt.
- **Usage pattern:** If `ms_client.available()` returns True, call `search_skills` with the task description, prepend the top 3 results to the planner prompt.

## 3. Web Research Tools

### Markdown Browser (mdwb) — Deterministic Web Capture
- **Module:** `src/tools/markdown_browser.py`
- **Singleton:** `from src.tools.markdown_browser import md_browser`
- **Purpose:** Deterministic headless-browser page capture producing clean Markdown output + provenance manifest. Replaces screenshot-only browsing with structured text + links.
- **Key methods:**
  - `md_browser.capture_url(url: str, out_dir: str) -> Optional[Dict]` — captures URL, returns `{"out_md": path, "manifest": path}` or None
  - `md_browser.available() -> bool` — check if `mdwb` binary is on PATH
- **Config flag:** `[FEATURES] MDWB_BINARY = ""` — set path to `mdwb` binary to enable
- **Called in** `src/agents/researcher/researcher.py` via `_capture_url_safe()`.
- **Safety note:** All `mdwb` output is ACIP-checked before use in prompts (treats external content as untrusted).

|----------|-------|-----------|
| Anthropic (Claude) | `src/llm/claude_client.py` | `API_KEYS.CLAUDE` |

| Google (Gemini) | `src/llm/gemini_client.py` | `API_KEYS.GEMINI` |

| Groq | `src/llm/groq_client.py` | `API_KEYS.GROQ` |
| Ollama (local) | `src/llm/ollama_client.py` | `API_ENDPOINTS.OLLAMA` |
| LM Studio (local) | `src/llm/lm_studio_client.py` | `API_ENDPOINTS.LM_STUDIO` |

**All LLM calls go through `LLM.inference(prompt, project_name)` which automatically:**
1. Runs ACIP check (if `ACIP_ENABLED`)
2. Uses sanitized prompt
3. Tracks token usage
4. Emits timing/warning events to frontend

## 8. Configuration

All config lives in `config.toml` (auto-created from `sample.config.toml` on first run).

Key sections:
- `[API_KEYS]` — provider API keys (Bing, Google, Claude, OpenAI, Gemini, Mistral, Groq, Netlify, SkillsMP)
- `[API_ENDPOINTS]` — provider API base URLs (Bing, Google, Ollama, LM Studio, OpenAI)
- `[STORAGE]` — paths for DB, screenshots, PDFs, projects, logs, repos
- `[LOGGING]` — `LOG_REST_API`, `LOG_PROMPTS`
- `[TIMEOUT]` — `INFERENCE` (seconds)
- `[FEATURES]` — `ACIP_ENABLED`, `DCG_ENABLED`, `MS_BINARY`, `MDWB_BINARY`, `SKILL_MANAGER_ENABLED`

## 13. Flywheel Integration Checklist

When adding a new capability, follow this pattern:
1. Add a wrapper/client module in `src/` under the appropriate subfolder
2. Add ACIP check on all externally-sourced inputs before using in prompts
3. Add DCG check on all OS-level commands before executing
4. Add a config flag in `[FEATURES]` so it can be toggled off
5. Register in this toolbox (`agents-toolbox.md`)
6. Update `beads/todo.md` and commit
